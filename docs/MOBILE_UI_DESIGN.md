# Farm-tner 랜딩페이지 모바일 UI 고도화 설계

## 1. 개요

- **목적**: 예비 청년농의 서비스 이해도를 높이기 위해 toss.im 스타일의 모바일 UI 컴포넌트 추가
- **참고**: [toss.im](https://toss.im) — 카드 기반, 간결한 타이포그래피, 세로 정보 흐름
- **추가 요소**: 탭(빠른 진입 포인트), 추천 정보 섹션

---

## 2. 현재 구조 분석

| 섹션 | 내용 | 모바일 최적화 |
|------|------|---------------|
| Hero | 헤드라인 + CTA 2개 | ✅ 버튼 세로 배치 |
| Problem | 예비 청년농 고민 3카드 | ✅ 카드 세로 |
| Solution | Farm-tner 해결 + 사용자 흐름 | ⚠️ 흐름 가로 → 세로 전환 |
| Value Prop | 기존 vs Farm-tner 비교 | ⚠️ 비교 레이아웃 단순 |
| Features | 핵심 기능 4카드 | ✅ 그리드 → 1열 |
| Social Proof | 인용 카드 | ✅ |
| CTA | 이메일 폼 | ✅ |
| Footer | 정보 | ✅ |

**문제점**: 사용자가 “무엇을 할 수 있는지”를 한눈에 파악하기 어렵고, 창농 관련 실질 정보(정책, 교육, 금융)가 부족함.

---

## 3. 추가 컴포넌트 설계

### 3.1 탭 (Quick Action Tabs)

**위치**: Hero 바로 아래 (Problem 섹션 앞)

**목적**: 핵심 행동(정보 보기, 준비 확인, 상담 예약, 더보기)을 한 화면에서 선택 가능하게

**UI 구성** (2×2 그리드, 토스 스타일):

| 탭 | 아이콘 | 라벨 | 앵커/동작 |
|----|--------|------|-----------|
| 1 | 📋 (클립보드) | 정보 보기 | `#recommended` (추천 정보 섹션) |
| 2 | ✅ (체크) | 준비 확인 | `#solution` (체크리스트 설명) |
| 3 | 🤝 (악수) | 상담 예약 | `#cta` (베타 신청 폼) |
| 4 | 💰 (금융/자산) | 더보기 | `#features` (핵심 기능) |

**스타일 가이드**:
- 카드: 흰 배경, 라운드 코너(16px), 연한 테두리/그림자
- 아이콘: 상단 중앙, 약 40px
- 라벨: 아이콘 아래, 14~16px
- 터치 영역: 최소 44×44px
- hover(데스크톱): 약한 그림자·배경 변화

---

### 3.2 추천 정보 (Recommended Information)

**위치**: 탭 섹션 아래, Problem 앞 또는 Solution 다음

**목적**: “실제로 어떤 혜택을 받을 수 있는지”를 구체적 예시로 전달

**UI 구성** (토스 스타일 카드 리스트):
- 제목: **추천 정보** (굵은 제목)
- 카드: 세로 스택, 각 카드에 아이콘 + 제목 + 설명 2줄

**예시 데이터 (MVP)**:

| 카드 | 제목 | 설명 1 | 설명 2 |
|------|------|--------|--------|
| 1 | 청년농 영농정착 지원 | 50,000,000원 지원 | 마감: 2024.12.31 |
| 2 | 스마트팜 기초 교육 | 온라인 교육 프로그램 | 시작: 11월 |
| 3 | 청년농 우대 대출 | 농협 연 2% 저금리 | 신청 가능 |

**스타일 가이드**:
- 제목: 진한 글자
- 설명 1: 회색
- 마감/기한: 강조(빨간색 등)
- 아이콘: 카드 왼쪽, 32~40px
- 카드: 흰 배경, 라운드 12px, 그리드/리스트 간격 12px

---

## 4. 섹션 재배치 제안

**변경 전**  
Hero → Problem → Solution → Value Prop → Features → Social Proof → CTA → Footer

**변경 후** (모바일 흐름 최적화):

1. **Hero** — 헤드라인 + CTA
2. **탭 (NEW)** — 정보 보기 / 준비 확인 / 상담 예약 / 더보기
3. **추천 정보 (NEW)** — 정책·교육·금융 예시 3개
4. **Problem** — 예비 청년농 고민
5. **Solution** — Farm-tner 해결 + 사용자 흐름
6. **Value Prop** — 기존 vs Farm-tner
7. **Features** — 핵심 기능 4개
8. **Social Proof** — 인용
9. **CTA** — 이메일 신청
10. **Footer**

탭을 클릭하면 해당 섹션으로 부드럽게 스크롤(anchor scroll).

## 5. toss.im 참고 포인트

| 요소 | toss.im | Farm-tner 적용 |
|------|---------|----------------|
| 카드 | 흰 배경, 둥근 모서리, 약한 그림자 | 동일 |
| 여백 | 넓은 패딩, 세로 간격 | 동일 유지 |
| 타이포 | 제목 굵게, 본문 회색 | 동일 |
| CTA | 명확한 버튼, 일관된 컬러 | Primary #2F855A 유지 |
| 정보 카드 | 아이콘 + 제목 + 2줄 설명 | 추천 정보 카드에 적용 |
| 그리드 | 모바일 1열, 데스크톱 2~4열 | 탭 2×2, 추천 정보 1열 |

---

## 6. 반응형 동작

| 화면 | 탭 | 추천 정보 |
|------|-----|-----------|
| 모바일 (<768px) | 2×2 그리드, 터치 친화 | 1열 세로 스택 |
| 태블릿 (768~1024px) | 2×2 또는 4열 | 1~2열 |
| 데스크톱 (>1024px) | 4열 가로 | 3열 그리드 또는 리스트 |

---

## 7. GA4 추적 (선택)

- 탭 클릭: `tab_click`, `event_label`: 정보보기 / 준비확인 / 상담예약 / 더보기
- 추천 정보 카드 클릭: `recommended_click`, `event_label`: 카드 제목

---

## 8. 구현 우선순위

| 단계 | 작업 | 예상 |
|------|------|------|
| 1 | 탭 섹션 추가 (HTML + CSS) | 1차 |
| 2 | 추천 정보 섹션 추가 (HTML + CSS) | 1차 |
| 3 | 섹션 ID 연결, 스크롤 동작 | 1차 |
| 4 | GA4 이벤트 추가 | 2차 |
| 5 | 데이터 동적화(JSON/API) | 3차 |

## 9. 산출물 요약

- **탭**: 4개 Quick Action 카드, 2×2 그리드
- **추천 정보**: 제목 + 3개 정보 카드 (정책·교육·금융)
- **레이아웃**: Hero 다음에 배치, 기존 컬러·타이포 유지
- **반응형**: 모바일 우선, 데스크톱에서 그리드 확장
